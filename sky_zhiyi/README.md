# SKY职忆｜AI求职记忆库与岗位适配平台

> 一次导入，多岗位适配。  
> 面向大学生求职场景的 AI 求职资产管理与智能岗位适配平台。

---

## 项目简介

**SKY职忆** 是一款面向大学生求职场景的 AI 求职记忆库与岗位适配平台。

项目围绕大学生在求职过程中常见的几个问题展开：

- 简历经历分散，项目、实习、竞赛、课程成果难以统一整理；
- 同一段经历面对不同岗位时需要反复改写，重复劳动高；
- 求职者不知道自己适合哪些岗位，岗位筛选效率低；
- 面试准备缺乏针对性，无法根据个人经历生成个性化追问；
- 面试反馈难以沉淀，下一次求职无法复用。

SKY职忆通过 **材料导入、经历精炼、求职记忆库、岗位适配、面试训练和反馈回写** 构建完整闭环，帮助学生把零散经历转化为可长期复用的求职资产。

---

## 项目定位

本项目不是单纯的 AI 简历生成器，也不是普通的面试聊天机器人，而是一个面向大学生求职全流程的 **AI 求职记忆中台**。

核心理念是：

> 将学生已有的项目、实习、竞赛、课程、技能和面试反馈沉淀为结构化求职记忆，并在不同岗位投递、简历表达和面试训练中长期复用。

---

## 核心功能

### 1. 用户材料导入

系统支持用户通过两种方式构建初始求职资料：

- **已有简历**：上传或粘贴简历内容；
- **暂无简历**：通过自然语言输入专业、年级、项目、实习、竞赛、技能和求职方向。

当前版本支持上传：

- `.txt`
- `.md`
- `.pdf`
- `.docx`

上传后，后端会读取文件正文，并进入材料解析流程。

---

### 2. 材料解析智能体

材料解析智能体用于从用户输入的简历或经历材料中提取结构化求职信息，包括：

- 教育背景；
- 目标岗位；
- 技能关键词；
- 项目经历；
- 实习经历；
- 竞赛经历；
- 缺失字段。

解析结果不会直接以大段代码形式展示，而是通过加载动画与关键词卡片逐步呈现，使用户能直观看到系统正在识别什么内容。

---

### 3. 经历精炼智能体

经历精炼智能体负责将解析结果进一步转化为可复用的求职记忆。

主要处理内容包括：

- 去除重复信息；
- 合并相似经历；
- 提炼能力标签；
- 判断经历的重要程度；
- 生成长期求职记忆；
- 写入个人求职记忆库。

示例能力标签包括：

- Python
- 数据分析
- 项目表达
- 科研写作
- 产品设计
- 跨学科能力
- 面试表达
- 复盘能力

---

### 4. 求职记忆库

求职记忆库用于保存用户长期可复用的结构化经历资产。

系统保存的不是普通聊天记录，而是经过精炼后的求职资产，例如：

- 项目经历记忆；
- 实习经历记忆；
- 技能标签；
- 面试反馈；
- 面试回答练习；
- 岗位相关经历。

求职记忆库中的内容可以被后续岗位适配、面试训练和回答评价重复调用。

---

### 5. 岗位适配

系统内置本地校招岗位库，支持根据用户输入的岗位关键词和城市偏好进行岗位匹配。

当前岗位适配模块支持：

- 岗位关键词检索；
- 城市筛选；
- 匹配度计算；
- 低匹配岗位过滤；
- 岗位卡片展示；
- 投递链接跳转。

系统只展示匹配度较高的岗位，避免向用户推荐大量无关岗位。

岗位卡片中重点展示：

- 公司名称；
- 企业类型；
- 匹配度；
- 招聘岗位；
- 工作地点；
- 是否笔试；
- 截止日期；
- 投递链接。

长岗位文本会默认折叠，避免界面拥挤。

---

### 6. 面试训练

系统可根据用户的求职记忆库和目标岗位生成个性化面试训练内容。

功能包括：

- 生成 1 分钟自我介绍；
- 生成通用面试问题；
- 生成岗位相关问题；
- 生成项目追问题；
- 支持用户输入回答；
- 对用户回答进行评价；
- 提供回答改进建议；
- 将面试回答和反馈保存到记忆库。

这使系统形成：

> 面试训练 → 用户回答 → 智能评价 → 反馈回写 → 下次复用

的闭环。

## 本地运行方式

### 1. 克隆项目

```bash
git clone https://github.com/lovesky514514-cmd/sky_zhiyi.git
cd sky_zhiyi
```

---

### 2. 配置 DeepSeek API Key

进入后端目录：

```text
backend/.env
```

将其中的：

```text
DEEPSEEK_API_KEY=把你的DeepSeek_API_Key粘贴到这里
```

修改为：

```text
DEEPSEEK_API_KEY=你的真实DeepSeek_API_Key
```

默认模型配置为：

```text
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_BASE_URL=https://api.deepseek.com/chat/completions
```

---

### 3. 安装后端依赖

进入 `backend` 文件夹：

```bash
cd backend
pip install -r requirements.txt
```

---

### 4. 启动后端

```bash
python server.py
```

如果是 Windows 系统，也可以直接双击：

```text
backend/run_backend.bat
```

启动成功后，终端会显示：

```text
Running on http://127.0.0.1:5000
```

---

### 5. 打开网页

在浏览器中访问：

```text
http://127.0.0.1:5000
```

注意：不要直接双击 `index.html`。项目需要通过 Flask 后端启动，否则岗位检索、文件上传和 DeepSeek 调用可能无法正常工作。

## DeepSeek 接入说明

项目通过后端代理调用 DeepSeek Chat 模型。

前端不会直接保存 API Key，避免密钥泄露。

调用流程如下：

```text
前端页面
   ↓
Flask 后端接口
   ↓
DeepSeek API
   ↓
返回结构化 JSON
   ↓
前端展示
```

主要接口包括：

```text
POST /api/parse
POST /api/refine
POST /api/interview
POST /api/evaluate
```

对应功能：

| 接口 | 功能 |
|---|---|
| `/api/parse` | 简历与经历材料解析 |
| `/api/refine` | 经历精炼与求职记忆生成 |
| `/api/interview` | 面试训练问题生成 |
| `/api/evaluate` | 用户面试回答评价 |

## 项目创新点

### 1. 从一次性生成工具到求职记忆库

传统 AI 求职工具多用于单次简历修改或面试问答，本项目强调将用户经历沉淀为可长期复用的结构化求职资产。

### 2. 一次导入，多岗位复用

用户只需导入一次简历或经历材料，系统即可在不同岗位投递、简历表达和面试训练中重复调用相关经历。

### 3. 多智能体求职流程

系统将求职过程拆解为多个智能体任务：

- 材料解析；
- 经历精炼；
- 岗位适配；
- 面试训练；
- 回答评价；
- 反馈回写。

相比单一聊天机器人，该流程更清晰，也更适合实际求职场景。

### 4. 面试反馈回写机制

用户在面试训练或真实面试后的反馈可以继续写入记忆库，使后续求职建议更加贴合用户实际情况。
